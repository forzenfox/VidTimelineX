# 页面布局规格文档

## 1. 文档信息

- **文档名称**：页面布局规格文档
- **版本**：1.0
- **最后更新**：2026-01-28
- **作者**：UI 设计团队
- **适用范围**：亿口甜筒·时光视频集应用

## 2. 整体布局结构

### 2.1 基础布局模式

应用采用响应式网格布局，基于 Tailwind CSS 的网格系统构建。整体布局分为以下几个主要区域：

- **页面头部**（Header）：包含应用标题、导航菜单、搜索框等
- **主内容区**（Main Content）：包含主要的内容展示
- **侧边栏**（Sidebar）：包含辅助信息、快捷操作等
- **页面底部**（Footer）：包含版权信息、链接等

### 2.2 布局容器

- **全局容器**：`max-w-[1440px] lg:max-w-[1600px] mx-auto px-6`
- **内容容器**：`flex flex-col md:flex-row gap-8`
- **侧边栏宽度**：`md:w-80 lg:w-96 shrink-0`

## 3. 页面布局规格

### 3.1 甜筒页面（TiantongPage）

**布局结构**：

1. **页面头部**：
   - 高度：自适应，默认约 100px
   - 背景：半透明卡片背景，带模糊效果
   - 定位：`sticky top-0 z-40`
   - 内容：左侧 Logo 和标题，右侧搜索框和主题切换按钮

2. **主内容区**：
   - 布局：`flex-1 w-full min-w-0`
   - 内容：视频时间线（VideoTimeline）

3. **侧边栏**：
   - 宽度：`md:w-80 lg:w-96 shrink-0`
   - 内容：侧边弹幕（SidebarDanmu）

4. **页面底部**：
   - 高度：约 100px
   - 背景：卡片背景
   - 内容：版权信息、主题标识等

5. **特殊元素**：
   - 水平弹幕：覆盖整个页面的固定定位元素
   - 页面边角装饰图标：固定定位的装饰元素

### 3.2 驴酱页面（LvjiangPage）

**布局结构**：

1. **页面头部**：
   - 高度：自适应
   - 内容：Logo、标题、主题切换按钮等

2. **主内容区**：
   - 布局：居中布局
   - 内容：视频时间线（VideoTimeline）、装饰元素

3. **侧边栏**：
   - 位置：页面底部
   - 内容：侧边弹幕（SideDanmaku）

4. **特殊元素**：
   - 水平弹幕：覆盖整个页面的固定定位元素
   - 装饰元素：猪纹路图案、背景渐变等

## 4. 组件布局规格

### 4.1 视频时间线（VideoTimeline）

- **布局**：垂直时间线布局
- **卡片间距**：`gap-6`
- **卡片宽度**：100%
- **响应式调整**：在不同屏幕尺寸下调整卡片大小和布局

### 4.2 侧边弹幕（SidebarDanmu/SideDanmaku）

- **布局**：垂直滚动布局
- **高度**：`h-[calc(100vh-180px)] sticky top-20`
- **内容结构**：顶部标签栏、提示信息、直播公告区、弹幕互动区、底部操作栏

### 4.3 水平弹幕（HorizontalDanmaku）

- **布局**：固定定位，覆盖整个页面
- **层级**：`z-31`（甜筒页面）/ `z-30`（驴酱页面）
- **动画**：从右向左滚动

### 4.4 视频模态框（VideoModal）

- **布局**：居中模态框
- **宽度**：`max-w-4xl`
- **高度**：自适应
- **层级**：`z-50`

## 5. 响应式布局断点

| 断点名称 | 屏幕宽度 | 布局调整 |
|---------|---------|----------|
| 移动端 | < 768px | 单列布局，侧边栏移至底部 |
| 平板 | 768px - 1024px | 双列布局，调整元素大小 |
| 桌面 | > 1024px | 完整双列布局，最大宽度限制 |

## 6. 布局规则与最佳实践

### 6.1 布局规则

1. **容器嵌套**：避免过深的容器嵌套，保持 DOM 结构简洁
2. **Flexbox 与 Grid**：优先使用 Flexbox 进行一维布局，Grid 进行二维布局
3. **间距一致性**：使用统一的间距系统，避免随意设置间距
4. **对齐方式**：保持元素对齐一致，使用 `items-center`、`justify-between` 等类

### 6.2 最佳实践

1. **使用语义化 HTML**：使用 `<header>`、`<main>`、`<aside>`、`<footer>` 等语义化标签
2. **响应式优先**：从移动端开始设计，逐步扩展到更大的屏幕
3. **性能优化**：避免使用过多的固定定位元素，减少重排重绘
4. **可访问性**：确保布局对屏幕阅读器友好，使用适当的 ARIA 属性

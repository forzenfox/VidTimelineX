## 2. 字体选择

### 2.1 主字体

- **字体家族**：`font-sans`（Tailwind CSS 无衬线字体）
- **具体字体**：
  - 系统默认无衬线字体
  - 回退字体链：`-apple-system, BlinkMacSystemFont, "Segoe UI", "PingFang SC", "Hiragino Sans GB", "Microsoft YaHei", sans-serif`
- **适用场景**：所有页面文本、标题、按钮等

### 2.2 特殊字体

- **等宽字体**：`font-mono`
  - 适用场景：数字、代码、特殊标识等
  - 示例：`12195609`（房间号）

## 3. 字体大小系统

### 3.1 基础字体大小

- **默认字体大小**：`16px`（浏览器默认）
- **根字体大小**：`1rem = 16px`

### 3.2 字体大小层级

| 尺寸名称 | 字体大小 | 适用场景 | 代码示例 |
|---------|---------|----------|----------|
| 极小 | `text-[10px]` | 小标签、徽章 | `text-[10px]` |
| 特小 | `text-xs` | 辅助文本、说明文字 | `text-xs` |
| 小 | `text-sm` | 次要文本、输入框占位符 | `text-sm` |
| 基础 | `text-base` | 正文、普通文本 | `text-base` |
| 中 | `text-lg` | 副标题、强调文本 | `text-lg` |
| 大 | `text-xl` | 标题、重要信息 | `text-xl` |
| 特大 | `text-2xl` | 主标题、页面标题 | `text-2xl` |
| 超大 | `text-3xl` | 页面主标题、品牌名称 | `text-3xl` |

### 3.3 响应式字体大小

| 断点 | 字体大小调整 | 适用类名 |
|------|-------------|----------|
| 移动端 (< 768px) | 默认字体大小 | 无特殊类名 |
| 平板 (768px - 1024px) | 字体大小略小 | `text-base` → `font-size: 0.95rem` |
| 桌面 (> 1024px) | 标准字体大小 | 无特殊类名 |

## 8. 排版最佳实践

### 8.1 通用规则

1. **保持一致性**：在整个应用中使用统一的排版系统
2. **可读性优先**：确保文本清晰易读，避免过小的字体或过密的行高
3. **响应式适配**：在不同屏幕尺寸下调整字体大小和布局
4. **层次分明**：使用不同的字体大小、字重和颜色创建清晰的视觉层次
5. **适当留白**：使用适当的间距和行高创造舒适的阅读体验

### 8.2 标题排版

1. **标题层级**：使用 H1-H6 标签创建清晰的标题层级
2. **字体大小**：从 H1 到 H6 逐渐减小字体大小
3. **字重**：标题使用粗体或黑体，与正文形成对比
4. **间距**：标题与正文之间保持适当的间距

### 8.3 正文排版

1. **行高**：正文使用 `leading-relaxed` 或 `leading-normal`
2. **段落间距**：段落之间保持适当的间距
3. **对齐方式**：正文使用左对齐，避免两端对齐
4. **行长度**：每行文字保持在 50-75 个字符之间，提高可读性

### 8.4 特殊元素排版

1. **按钮文本**：使用中等字重，适当的字体大小和字间距
2. **输入框文本**：使用标准字体大小，确保清晰易读
3. **标签和徽章**：使用小字体，紧凑的行高和字间距
4. **链接文本**：使用主题强调色，悬停时添加下划线

# 响应式设计适配策略

## 1. 支持的设备范围

本项目**明确放弃对移动端设备（屏幕宽度小于768px的设备）的适配支持**，同时保留并优化对以下设备的适配能力：

| 设备类型 | 屏幕宽度范围 | 示例设备 |
|---------|------------|---------|
| 平板设备 | 768px - 1024px | iPad, 小屏平板 |
| 桌面设备 | 1024px 及以上 | 笔记本电脑, 台式电脑, 大屏显示器 |

## 2. 屏幕尺寸标准

### 2.1 平板设备 (Tablet)
- **最小宽度**: 768px
- **最大宽度**: 1024px
- **推荐分辨率**: 768x1024, 1024x768
- **布局特点**: 单列或双列布局，组件尺寸适中

### 2.2 桌面设备 (Desktop)
- **最小宽度**: 1024px
- **推荐分辨率**: 1024x768, 1366x768, 1440x900, 1920x1080 及以上
- **布局特点**: 双列或多列布局，组件尺寸较大，空间利用率高

## 3. 适配策略变更内容

### 3.1 设备检测逻辑调整
- 修改了 `src/hooks/use-mobile.ts` 中的设备检测逻辑
- 移除了移动端设备类型判断
- 调整了断点定义，最小支持宽度为768px
- `useIsMobile()` 函数现在始终返回 `false`

### 3.2 动态组件加载调整
- 更新了 `src/hooks/use-dynamic-component.tsx`
- 移除了移动端组件配置选项
- 优化了平板和桌面端的组件加载逻辑

### 3.3 样式规则优化
- 修改了 `src/index.css`
- 移除了针对移动端的媒体查询规则
- 保留并优化了平板和桌面端的样式规则
- 添加了平板设备特定的样式优化

### 3.4 页面组件调整
- 更新了 `src/pages/Home.tsx`
- 移除了移动端特定的 `MobileSidebar` 组件
- 移除了移动端搜索按钮和搜索栏
- 调整了搜索框的显示逻辑，现在在所有支持的设备上均可见

## 5. 开发和维护指南

### 5.1 媒体查询使用
- 优先使用 Tailwind CSS 的响应式类
- 如需自定义媒体查询，参考以下断点：
  - 平板设备: `max-width: 1024px`
  - 桌面设备: `min-width: 1025px`

### 5.2 组件设计原则
- 确保组件在 768px 及以上屏幕尺寸下正常显示
- 避免使用固定宽度，优先使用相对单位和弹性布局
- 考虑不同屏幕尺寸下的组件交互方式

### 5.3 性能优化
- 针对平板和桌面设备优化图片和资源加载
- 考虑大屏幕设备的性能需求

## 6. 变更原因

- **专注核心用户群体**: 分析显示大部分用户使用平板和桌面设备访问
- **简化开发流程**: 减少移动端适配的开发和维护成本
- **优化用户体验**: 专注于提升平板和桌面端的用户体验
- **明确技术债务**: 清晰定义项目的设备支持范围，便于后续维护和扩展

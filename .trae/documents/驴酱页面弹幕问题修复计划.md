## 🔍 问题分析

### 对比甜筒页面和驴酱页面的实现差异

**甜筒页面** ([HorizontalDanmaku.tsx](file:///d:/workspace/VidTimelineX/frontend/src/features/tiantong/components/HorizontalDanmaku.tsx))：
```tsx
// 直接读取文本文件
import danmakuText from "../data/danmaku.txt?raw";
const textPool = useMemo(() => {
  return danmakuText.split('\n').filter(line => line.trim());
}, [danmakuText]);

// 生成弹幕列表
textPool.length = 61 条弹幕
for (let i = 0; i < textPool.length; i++) { ... }
```

**驴酱页面** ([HorizontalDanmaku.tsx](file:///d:/workspace/VidTimelineX/frontend/src/features/lvjiang/components/HorizontalDanmaku.tsx))：
```tsx
// 从 JSON 导入数据
import { dongzhuDanmaku, kaigeDanmaku, commonDanmaku } from "../data";
const pool = theme === "dongzhu"
  ? [...dongzhuDanmaku, ...commonDanmaku]
  : [...kaigeDanmaku, ...commonDanmaku];

// 使用 DanmakuGenerator 生成
const generator = new DanmakuGenerator({
  theme: theme as "dongzhu" | "kaige",
  textPool: pool,  // 61 条弹幕
  ...
});

const messages = generator.generateBatch({
  count: 20,  // 只生成 20 条！
  type: "horizontal",
  ...
});
```

### 问题根因

1. **弹幕数量不足**：驴酱页面只生成 20 条弹幕，而甜筒页面生成 61 条
2. **轨道位置问题**：`getRandomTop()` 方法返回 `trackIndex / trackCount`，即 0/8, 1/8, 2/8...，但弹幕容器设置了 `top: 80px`，导致弹幕位置计算可能有问题
3. **延迟设置问题**：驴酱页面的 delay 是 `index * 300` 毫秒，而甜筒页面是 `i * 0.2` 秒（200ms）
4. **duration 单位问题**：驴酱页面 `duration: (msg.duration || 8) * 1000` 将秒转为毫秒，但动画 CSS 可能期望的是秒

### 关键差异对比表

| 项目 | 甜筒页面 | 驴酱页面 |
|------|----------|----------|
| 弹幕数量 | 61 条 | 20 条 |
| top 值 | `10 + trackIndex * 10` (%) | `trackIndex / trackCount` (0-1) |
| delay | `i * 0.2` (秒) | `index * 300` (毫秒) |
| duration | `6-10` 秒 | `8000-12000` 毫秒 |
| 容器样式 | `fixed inset-0` | `fixed top: 80px` |

## 🎯 修复方案

参照甜筒页面实现，修改驴酱页面的 HorizontalDanmaku 组件：

### 方案 1：简化实现（推荐）

完全参照甜筒页面的实现方式，不使用 DanmakuGenerator，直接处理弹幕数据：

```tsx
// 直接导入 JSON 数据
import danmakuData from "../data/danmaku.json";

// 简化生成逻辑
const danmakuList = useMemo(() => {
  if (!isVisible) return [];
  
  const pool = theme === "dongzhu"
    ? [...danmakuData.dongzhuDanmaku, ...danmakuData.commonDanmaku]
    : [...danmakuData.kaigeDanmaku, ...danmakuData.commonDanmaku];
  
  const trackCount = 8;
  
  return pool.map((text, i) => ({
    id: `danmaku-${i}`,
    text,
    top: 10 + (i % trackCount) * 10,  // 10%, 20%, 30%...
    delay: i * 0.2,  // 秒
    duration: 6 + Math.random() * 4,  // 秒
  }));
}, [theme, isVisible]);
```

### 方案 2：修复现有实现

保持使用 DanmakuGenerator，但修复以下问题：
1. 增加生成数量到 61 条
2. 修复 top 值计算
3. 统一 duration 单位为秒

# 统一弹幕实现优化计划

**创建时间：** 2026-02-28
**目标：** 统一甜筒页面和驴酱页面的飘屏弹幕实现方案

---

## 📋 当前实现分析

### 甜筒页面 (Tiantong)

**HorizontalDanmaku.tsx 特点：**
- ✅ 使用 `danmakuKey` 机制强制重新渲染
- ❌ `durationMap` 使用 `useState`，只在首次渲染初始化
- ❌ 弹幕 ID 固定为 `danmaku-${i}`，不包含主题信息
- ❌ 缺少 `isVisible` 控制

### 驴酱页面 (Lvjiang)

**HorizontalDanmaku.tsx 特点：**
- ✅ 使用 `danmakuKey` 机制强制重新渲染
- ❌ `durationMap` 使用 `useState`，只在首次渲染初始化
- ❌ 弹幕 ID 固定为 `danmaku-${i}`，不包含主题信息
- ✅ 有 `isVisible` 控制

### 问题总结

1. **durationMap 缓存问题**：两个页面都使用 `useState` 缓存 durationMap，但主题切换时不会重新生成
2. **弹幕 ID 不唯一**：固定 ID 可能导致 React 无法正确识别弹幕变化
3. **实现不一致**：甜筒页面缺少 `isVisible` 控制

---

## 🎯 优化方案

### 方案选择：统一使用 yuxiaoc 简化方案

**核心改动：**
1. **durationMap 改用 useMemo** - 随主题变化重新生成
2. **弹幕 ID 包含主题信息** - `danmaku-${theme}-${i}`
3. **可选：移除 danmakuKey 机制** - 如果弹幕组件优化后

**优化后的实现：**

```tsx
export function HorizontalDanmaku({ theme, isVisible = true }: HorizontalDanmakuProps) {
  // durationMap 放入 useMemo，随 theme 变化重新生成
  const durationMap = useMemo(() => {
    const map = new Map<number, number>();
    const pool = getDanmakuPool(theme);
    for (let i = 0; i < pool.length; i++) {
      map.set(i, 6 + Math.random() * 4);
    }
    return map;
  }, [theme]);

  const danmakuList = useMemo(() => {
    if (!isVisible) return [];
    
    const pool = getDanmakuPool(theme);
    const trackCount = 8;
    
    return pool.map((text, i) => ({
      id: `danmaku-${theme}-${i}`, // ID 包含主题，确保唯一性
      text,
      top: 10 + (i % trackCount) * 10,
      delay: i * 0.2,
      duration: durationMap.get(i) || 8,
      color: getThemeColor(theme),
    }));
  }, [theme, isVisible, durationMap]);
  
  // ...
}
```

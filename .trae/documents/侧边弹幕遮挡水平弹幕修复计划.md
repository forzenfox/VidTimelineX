# 侧边弹幕栏遮挡水平飘屏弹幕 - 修复计划（深入分析版）

## 问题根因分析

### 1. 当前 z-index 层级关系（甜筒页面）

| 组件 | z-index | Position | 文件位置 |
|------|---------|----------|----------|
| 导航栏 Header | `z-40` | fixed | TiantongPage.tsx:L236 |
| 侧边弹幕 SidebarDanmu | `z-30` | sticky/fixed | SidebarDanmu.tsx:L98 |
| 水平飘屏弹幕 HorizontalDanmaku | `z-30` | fixed | HorizontalDanmaku.tsx:L64 |

### 2. 问题原因

将水平飘屏弹幕的 z-index 从 `z-40` 降低到 `z-30` 后，与侧边弹幕栏的 z-index 相同。由于侧边弹幕栏在 DOM 中位于水平弹幕之后，导致侧边弹幕栏遮挡了水平飘屏弹幕。

## 深入分析：降低侧边弹幕 z-index 的影响

### 甜筒页面完整 z-index 层级表

| 层级 | 组件 | 用途 | 风险等级 |
|------|------|------|----------|
| **z-50** | Dialog/Sheet/Drawer/Tooltip/VideoModal/LIVE标签 | 弹窗、抽屉、工具提示 | 无影响（始终最高） |
| **z-40** | 导航栏 Header | 固定顶部导航 | 无影响（位于顶部不重叠） |
| **z-30** | 水平飘屏弹幕 HorizontalDanmaku | 全屏飘屏弹幕 | **关键组件** |
| **z-20** | yuxiaoc/lvjiang 侧边弹幕 | 其他页面侧边栏 | 参考实现 |
| **z-10** | 视频卡片、工具栏、时间轴按钮 | 页面内容元素 | **潜在风险** |

### 降低侧边弹幕到 z-20 的潜在问题分析

#### ✅ 无影响的组件

| 组件 | 当前z-index | 说明 |
|------|-------------|------|
| 视频弹窗 VideoModal | z-50 | 弹窗层级远高于侧边弹幕，无影响 |
| 下拉菜单 DropdownMenu | z-50 | 菜单层级高，无影响 |
| 移动端抽屉 Sheet/Drawer | z-50 | 抽屉层级高，无影响 |
| 工具提示 Tooltip | z-50 | 提示层级高，无影响 |
| 导航栏 Header | z-40 | 位于页面顶部，与侧边弹幕不重叠 |

#### ⚠️ 需要关注的问题

| 组件 | 当前z-index | 潜在影响 | 解决方案 |
|------|-------------|----------|----------|
| **水平飘屏弹幕** | z-30 | 会遮挡侧边弹幕 | **这是预期行为** |
| 视频卡片悬停效果 | z-10 | 可能遮挡侧边弹幕边缘 | 可接受 |
| 时间轴播放按钮 | z-10 | 可能遮挡侧边弹幕边缘 | 可接受 |
| 工具栏 IconToolbar | z-10 | 可能遮挡侧边弹幕边缘 | 可接受 |

### 核心问题：水平弹幕与侧边弹幕的显示策略

**关键发现**：

1. **水平弹幕** (`HorizontalDanmaku`) 使用 `fixed inset-0` + `pointer-events-none` 全屏覆盖
   - 目的是在全屏范围内飘动弹幕
   - `pointer-events-none` 确保不影响页面交互
   - 设置 `top: 80px` 避开导航栏

2. **侧边弹幕** (`SidebarDanmu`) 位于页面右侧固定区域
   - 桌面端：`sticky` 定位，宽度约 320px-384px
   - 移动端：`fixed` 定位，底部抽屉形式

3. **两者的关系**：
   - 水平弹幕需要**覆盖整个视口**（除导航栏外）
   - 侧边弹幕是**局部区域**的固定内容
   - **水平弹幕应该在侧边弹幕之上**，因为飘屏弹幕是临时飘过的，侧边弹幕是固定内容

### 与其他页面的对比验证

| 页面 | 水平弹幕 | 侧边弹幕 | 实现方式 | 效果 |
|------|---------|---------|----------|------|
| **余小C** | z-30 | z-20 | 侧边弹幕 z-20，水平弹幕 z-30 | ✅ 水平弹幕在侧边弹幕之上 |
| **驴酱** | z-31 | z-20 | 侧边弹幕 z-20，水平弹幕 z-31 | ✅ 水平弹幕在侧边弹幕之上 |
| **甜筒（当前）** | z-30 | z-30 | 同级，DOM顺序决定 | ❌ 侧边弹幕遮挡水平弹幕 |
| **甜筒（修复后）** | z-30 | z-20 | 侧边弹幕 z-20，水平弹幕 z-30 | ✅ 水平弹幕在侧边弹幕之上 |

### 移动端特殊考虑

甜筒页面的侧边弹幕在移动端有两种状态：

1. **默认状态**：底部固定栏 (z-30)
2. **展开状态**：底部抽屉 (需要高层级)

**分析**：
- 移动端侧边弹幕默认是底部固定栏形式
- 水平弹幕在移动端同样使用 `top: 80px` 避开导航栏
- 降低侧边弹幕到 z-20 后，水平弹幕 (z-30) 会正确显示在侧边弹幕之上
- 这与余小C、驴酱的移动端实现一致

## 修复方案（修订版）

### 推荐方案：降低侧边弹幕 z-index 至 z-20

**修改文件**: `frontend/src/features/tiantong/components/SidebarDanmu.tsx`

**具体修改**:
```tsx
// 修改前 (第98行)
className={`${isMobile ? "fixed bottom-0 left-0 right-0 h-64 z-30 border-t border-b" : "h-[calc(100vh-180px)] sticky top-20 z-30"} ...`}

// 修改后
className={`${isMobile ? "fixed bottom-0 left-0 right-0 h-64 z-20 border-t border-b" : "h-[calc(100vh-180px)] sticky top-20 z-20"} ...`}
```

### 修复后的层级关系

```
z-50: 弹窗、抽屉、工具提示 (最高层级，临时覆盖)
z-40: 导航栏 Header (固定顶部)
z-30: 水平飘屏弹幕 HorizontalDanmaku (全屏飘动)
z-20: 侧边弹幕 SidebarDanmu (固定侧边栏)
z-10: 页面内容、视频卡片、工具栏
```

### 预期效果

1. **导航栏** (z-40) 始终在最上层，不被任何弹幕遮挡 ✅
2. **水平飘屏弹幕** (z-30) 在侧边弹幕之上，可以正常飘过侧边栏区域 ✅
3. **侧边弹幕** (z-20) 在水平弹幕之下，不会遮挡飘过的弹幕 ✅
4. **弹窗/抽屉** (z-50) 打开时覆盖所有内容 ✅

| 风险 | 可能性 | 影响 | 缓解措施 |
|------|--------|------|----------|
| 水平弹幕遮挡侧边弹幕内容 | 高 | 低 | **这是预期行为**，飘屏弹幕是临时的，不影响侧边栏交互 |
| 侧边弹幕被页面内容遮挡 | 低 | 中 | 页面内容使用 z-10，侧边弹幕 z-20 仍然高于它们 |

| 用户交互受影响 | 低 | 高 | 水平弹幕使用 `pointer-events-none`，不影响点击 |

## 结论

**降低侧边弹幕到 z-20 是安全的**，理由：

1. **与项目其他页面保持一致**：余小C和驴酱都使用 z-20 侧边弹幕 + z-30 水平弹幕的模式
2. **层级关系清晰**：导航栏 > 水平弹幕 > 侧边弹幕 > 页面内容
3. **不影响交互**：水平弹幕使用 `pointer-events-none`，不会阻挡用户操作
4. **无功能风险**：弹窗、抽屉等覆盖层使用 z-50，不受此修改影响
5. **视觉体验更好**：飘屏弹幕可以完整显示，不会被侧边栏截断

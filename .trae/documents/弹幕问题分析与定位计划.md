# 弹幕问题分析与定位计划

## 问题概述

1. **甜筒页面**：切换主题后，不会触发飘屏弹幕
2. **驴酱页面**：飘屏弹幕文字不明显（透明度或颜色问题）
3. **yuxiaoc 页面**：飘屏弹幕不触发

## 分析步骤

### 步骤 1：环境准备

* 启动 Playwright 浏览器

* 准备三个页面的测试 URL

### 步骤 2：甜筒页面分析（问题 1）

**目标**：定位切换主题后飘屏弹幕不触发的原因

**操作**：

1. 访问甜筒页面
2. 打开开发者工具，监听网络请求和控制台日志
3. 触发主题切换操作
4. 检查 HorizontalDanmaku 组件是否重新渲染
5. 检查弹幕数据是否正确加载
6. 检查是否有 JavaScript 错误
7. 对比驴酱页面的实现差异

**预期发现**：

* 主题切换时是否触发了弹幕组件的更新

* 弹幕数据是否正确传递

* 组件生命周期是否正确

### 步骤 3：驴酱页面分析（问题 2）

**目标**：定位飘屏弹幕文字不明显的原因

**操作**：

1. 访问驴酱页面
2. 定位飘屏弹幕元素
3. 检查弹幕文字的 CSS 样式：

   * color 属性（文字颜色）

   * opacity 属性（透明度）

   * text-shadow（文字阴影）

   * font-weight（字体粗细）
4. 检查背景颜色与文字颜色的对比度
5. 截图分析视觉效果

**预期发现**：

* 文字颜色是否与背景对比度不足

* 透明度过高导致文字不明显

* 缺少文字阴影等增强可读性的样式

### 步骤 4：yuxiaoc 页面分析（问题 3）

**目标**：定位飘屏弹幕不触发的原因

**操作**：

1. 访问 yuxiaoc 页面
2. 检查 HorizontalDanmaku 组件是否渲染
3. 检查弹幕数据是否正确加载
4. 检查组件的显示条件
5. 检查是否有 JavaScript 错误
6. 对比其他页面的实现差异

**预期发现**：

* 组件是否被正确导入和使用

* 弹幕数据是否为空或格式错误

* 是否有条件渲染逻辑阻止了显示

### 步骤 5：生成分析报告

**内容**：

* 每个问题的根本原因

* 修复建议

* 优先级排序

## 工具使用

* **Playwright**: 浏览器自动化、截图、元素检查

* **开发者工具**: 控制台日志、网络监控、样式检查

* **代码对比**: 三个页面实现差异对比

## 交付物

1. 问题分析报告
2. 截图证据
3. 修复建议代码

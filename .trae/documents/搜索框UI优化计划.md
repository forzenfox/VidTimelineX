# 搜索框UI优化计划

## 问题分析

根据用户反馈和截图分析，当前搜索框存在以下问题：

1. **搜索历史和清除按钮导致工具栏高度变化** - 当前搜索状态显示在展开框内部，导致整体高度增加，影响布局稳定性
2. **图标位置不对** - 放大镜图标和叉叉按钮位置不符合用户习惯

## 优化方案

### 方案一：采用类似筛选下拉框的悬浮设计（推荐）

参考 FilterDropdown 的实现方式，将搜索历史和清除功能放入悬浮框中，保持搜索框本身简洁。

**具体改动：**

#### 1. SearchButton 组件重构

**当前问题：**
- expanded 模式下搜索状态显示在输入框上方，增加高度
- 放大镜图标在输入框左侧，叉叉按钮在输入框内右侧

**优化后设计：**
```
┌─────────────────────────────────────┐
│ 🔍 搜索...                    ▼     │  ← 紧凑的触发按钮/输入框
└─────────────────────────────────────┘
           ↓ 点击展开悬浮框
┌─────────────────────────────────────┐
│ 🔍 甜筒                        ✕    │  ← 输入框（放大镜固定左侧，叉叉固定右侧）
├─────────────────────────────────────┤
│ 💡 搜索建议                         │
│ ┌────┐ ┌────┐ ┌────┐               │
│ └────┘ └────┘ └────┘               │
├─────────────────────────────────────┤
│ 🕐 搜索历史              [清空]     │
│ 甜筒                                │
│ 余小C                               │
└─────────────────────────────────────┘
```

**关键改动点：**
- 使用 PopoverPrimitive.Portal 将内容渲染到 body，避免影响父容器高度
- 输入框内：放大镜固定在左侧，叉叉按钮固定在右侧（无论是否有内容都显示占位）
- 搜索状态（如"搜索: 甜筒"）显示在悬浮框顶部或作为输入框的 placeholder
- 清除按钮改为"重置"按钮放在悬浮框底部或历史记录标题栏

#### 2. 文件修改清单

**SearchButton.tsx：**
1. 将 expanded 模式改为使用 Popover 悬浮框
2. 输入框内放大镜图标固定在左侧
3. 叉叉按钮改为始终显示（无内容时透明/禁用），固定在输入框最右侧
4. 搜索历史和建议放入悬浮框
5. 添加"重置"按钮清空搜索
6. 使用 PopoverPrimitive.Portal 避免高度影响

**VideoViewToolbar.tsx：**
1. 移除 SearchButton 外层的 flex-shrink-0 div（如果不再需要）
2. 保持现有布局不变

**IconToolbar.tsx：**
1. 保持搜索状态显示（因为 IconToolbar 在移动端和PC端都需要显示当前搜索状态）
2. 搜索按钮使用 icon 模式触发悬浮框

### 方案二：保持展开模式但优化布局

如果不希望使用悬浮框，可以保持 expanded 模式但优化布局：

1. **搜索状态外移** - 将"搜索: xxx"显示移到工具栏其他位置，而不是在搜索框内部
2. **固定高度** - 无论是否有搜索状态，都保持固定高度
3. **图标位置调整** - 按照用户要求调整放大镜和叉叉位置

**缺点：** 仍然会占用工具栏空间，不如悬浮框简洁

## 推荐方案：方案一

理由：
1. 与现有筛选下拉框设计保持一致，用户体验统一
2. 悬浮框不占用工具栏空间，布局更稳定
3. 搜索历史和清除功能在悬浮框内更合理
4. 代码复用性高，可以参考 FilterDropdown 的实现

- 放大镜和叉叉图标位置符合用户习惯
- 整体UI与筛选下拉框风格统一

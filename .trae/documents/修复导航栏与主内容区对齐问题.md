## 问题分析

右侧有一个固定宽度的弹幕栏（320px），导致：
1. 导航栏是 `fixed` 定位，`w-full` 覆盖整个视口
2. 导航栏内部的 `max-w-7xl` 容器在整个视口中居中
3. 主内容区实际可用宽度 = 视口宽度 - 320px（弹幕栏）
4. 因此导航栏和内容区的左右边界不对齐

## 修复方案

### 方案1：导航栏右侧留出弹幕栏空间（推荐）
修改导航栏容器的右边距，为弹幕栏留出空间：
- 在桌面端（lg及以上），导航栏容器右侧增加 `lg:pr-[320px]`
- 这样导航栏内容区与主内容区宽度一致，边界对齐

### 方案2：统一使用 calc 计算宽度
- 导航栏和内容区都使用 `max-w-[calc(100vw-320px)]`
- 但这会失去 max-width 的限制

### 方案3：使用 CSS Grid 布局
- 整体使用 Grid 布局，右侧固定 320px，左侧自适应
- 导航栏和内容区都在左侧区域内

## 采用方案1

修改 `Header.tsx`：
- 第111行：`<div className="max-w-7xl mx-auto px-4 sm:px-6 lg:px-8">`
- 改为：`<div className="max-w-7xl mx-auto px-4 sm:px-6 lg:px-8 lg:pr-[336px]">`

这样：
- 移动端/平板：正常全宽显示
- 桌面端：右侧留出320px（弹幕栏）+ 16px（间隙）空间

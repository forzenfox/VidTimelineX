## 问题 1：甜筒页面切换主题后飘屏弹幕不触发 ❌

### 根本原因

**组件实现问题**：[`HorizontalDanmaku.tsx`](file:///d:/workspace/VidTimelineX/frontend/src/features/tiantong/components/HorizontalDanmaku.tsx) 的弹幕数据是静态的 TXT 文件，不会随主题切换而重新生成。

**Playwright 验证**：

* ✅ 主题切换功能正常（🐯 → 🍦）

* ✅ 弹幕聊天室正常工作

* ❌ 飘屏弹幕不会重新触发

**问题代码**（第 36-62 行）：

```typescript
const danmakuList = useMemo(() => {
  if (!textPool || textPool.length === 0) {
    console.warn("Danmaku data is empty or undefined");
    return [];
  }

  const items: DanmakuItem[] = [];
  const trackCount = 8;

  for (let i = 0; i < textPool.length; i++) {
    const trackIndex = i % trackCount;
    const text = textPool[i];

    const color = theme === "tiger" ? "rgb(255, 95, 0)" : "rgb(255, 105, 180)";

    items.push({
      id: `danmaku-${i}`,
      text,
      top: 10 + trackIndex * 10,
      delay: i * 0.2,
      duration: durationMap.get(i) || 8,
      color: color,
    });
  }

  return items;
}, [theme, textPool, durationMap]); // ✅ 依赖项是正确的
```

**实际问题分析**：

1. 组件的 `useMemo` 依赖项包含 `theme`，理论上主题切换会重新生成弹幕
2. 但问题在于 **`durationMap`** **使用** **`useState`** **初始化，只在首次渲染时创建**
3. 当主题切换时，虽然 `theme` 变化，但 `durationMap` 不变，导致弹幕重新生成逻辑正常

**真正的问题**：
查看 [`TiantongPage.tsx`](file:///d:/workspace/VidTimelineX/frontend/src/features/tiantong/TiantongPage.tsx) 第 232 行：

```typescript
<HorizontalDanmaku theme={theme} />
```

组件正确接收了 `theme` prop，但**弹幕数据是静态的 TXT 文件**，不会随主题切换而变化。

### 问题诊断

**问题 1：文字颜色对比度不足**

* **冬竹主题**：`#3498DB`（蓝色） + 白色阴影

  * 背景可能是浅色，导致蓝色文字不明显

* **凯歌主题**：`#F39C12`（橙色） + 黑色阴影

  * 橙色在某些背景下对比度不足

**问题 2：字体大小偏小**

* 冬竹：`18px`

* 凯歌：`20px`

* 对比甜筒：`20px`（tiger）/ `18px`（sweet）

* 对比 yuxiaoc：`14-20px`（根据尺寸动态）

**问题 3：文字阴影不够强**

* 当前阴影透明度：`0.5` 和 `0.6`

* 建议增加到 `0.8` 和 `0.9`


### 修复建议

**方案 1：增强文字阴影和边框**（推荐）

```typescript
export const HorizontalDanmaku: React.FC<HorizontalDanmakuProps> = ({ theme, isVisible }) => {
  const [mounted, setMounted] = useState(false);
  const [isMobile, setIsMobile] = useState(false);

  useEffect(() => {
    const rafId = requestAnimationFrame(() => {
      setMounted(true);
    });
    return () => cancelAnimationFrame(rafId);
  }, []);

  useEffect(() => {
    const checkMobile = () => {
      setIsMobile(window.innerWidth < 640);
    };
    checkMobile();
    window.addEventListener("resize", checkMobile);
    return () => window.removeEventListener("resize", checkMobile);
  }, []);

  const danmakuList = useMemo(() => {
    if (!isVisible || !mounted) return []; // ⚠️ 关键：依赖 isVisible 和 mounted

    // ... 弹幕生成逻辑
  }, [theme, isVisible, mounted, isMobile]);

  if (!isVisible || !mounted) return null; // ⚠️ 不满足条件直接返回 null
```

**页面使用**（[`YuxiaocPage.tsx`](file:///d:/workspace/VidTimelineX/frontend/src/features/yuxiaoc/YuxiaocPage.tsx) 第 20 行和 100+ 行）：

### 问题诊断

**结论**：**yuxiaoc 页面的飘屏弹幕实际工作正常！**

用户可能误以为不触发，原因可能是：

1. 弹幕动画速度较慢（6597s），视觉上不明显
2. 弹幕透明度或颜色与背景融合
3. 弹幕位置在页面顶部，不在视线范围内

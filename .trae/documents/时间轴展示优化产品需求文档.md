# VidTimelineX 时间轴展示优化产品需求文档

## 一、项目背景与现状分析

### 1.1 产品定位

VidTimelineX 是一个视频收藏展示网站，主要用于展示用户收藏的B站视频内容。当前产品包含三个主要页面：tiantong（约50个视频）、lvjiang（约62个视频）、yuxiaoc（约3个视频）。

### 1.2 现有时间轴展示方式

* **桌面端**：时光轴左右交替式布局，视频卡片在轴线两侧交替排列

* **移动端**：单列垂直列表布局

* 每个视频卡片占用空间较大，页面纵向较长

### 1.3 当前布局示意

```
        [视频1]
            ↓
[视频2] ←━━━→ [视频3]
            ↓
        [视频4]
            ↓
[视频5] ←━━━→ [视频6]
```

## 二、用户痛点分析

### 2.1 核心痛点

随着视频数量增加（目前最多62个），用户面临以下问题：

| 痛点         | 描述              | 影响程度 |
| ---------- | --------------- | ---- |
| **浏览效率低**  | 需要大量滚动才能浏览完所有视频 | 🔴 高 |
| **难以快速定位** | 无法快速找到感兴趣的内容    | 🔴 高 |
| **视觉疲劳**   | 长时间浏览单调的纵向布局    | 🟡 中 |
| **信息密度低**  | 单页展示内容有限        | 🟡 中 |
| **选择困难**   | 一次性展示过多选项       | 🟡 中 |

### 2.2 用户行为分析

* 用户通常带着特定目标访问（如找某个UP主的视频）

* 用户可能想快速浏览最新/最热门的视频

* 用户习惯快速滑动浏览，决策时间短

### 2.3 需求优先级判定（KANO模型）

| 需求                | 分类   | 优先级 |
| ----------------- | ---- | --- |
| 视图模式切换（时间轴/网格/列表） | 期望需求 | P0  |
| 视频筛选/排序功能         | 期望需求 | P0  |
| 搜索功能增强            | 魅力需求 | P1  |
| 视频预览（悬停播放）        | 魅力需求 | P1  |
| 分页/无限滚动           | 基本需求 | P0  |

## 三、解决方案设计

### 3.1 视图模式切换

**提供三种视图模式：**

1. **时光轴模式（当前）**

   * 保留原有左右交替布局

   * 适合回忆浏览，强调时间顺序

   * 单页约展示8-12个视频

2. **网格模式（新增）**

   * 瀑布流/网格布局

   * 单页可展示20-40个视频

   * 适合快速浏览大量内容

3. **列表模式（新增）**

   * 紧凑型卡片列表

   * 显示更多视频信息（标题、UP主、时长、日期）

   * 单页可展示15-25个视频

### 3.2 视图模式对比

| 模式  | 单页展示量  | 适用场景      | 视觉特点 |
| --- | ------ | --------- | ---- |
| 时光轴 | 8-12个  | 回忆浏览、情感体验 | 强    |
| 网格  | 20-40个 | 快速浏览、发现内容 | 中    |
| 列表  | 15-25个 | 精准查找、信息对比 | 弱    |

### 3.3 筛选与排序功能

**筛选维度：**

* 发布时间（最近一周、一月、一年、自定义）

* UP主/标签

**排序方式：**

* 发布时间（最新/最早）

* 观看量（高/低）

* 收藏时间（最新/最早）

### 3.4 交互流程优化

```
用户进入页面
    ↓
默认显示「上次使用的视图模式」（记忆用户偏好）
    ↓
提供视图切换器（时光轴/网格/列表）
    ↓
可配合筛选/排序使用
    ↓
点击视频 → 打开详情/播放
```

## 四、功能需求详情

### 4.1 视图切换组件

**位置：** 时间轴标题下方工具栏

**UI设计：**

```
[时光轴] [网格] [列表]  |  [筛选▼] [排序▼] [🔍搜索]
```

**技术要求：**

* 视图切换使用本地存储记忆用户偏好

* 切换动画：300ms淡入淡出过渡

* 保留当前滚动位置（切换视图后）

### 4.2 网格模式规格

**桌面端：**

* 列数：4列（屏幕宽度>1200px）

* 列数：3列（屏幕宽度>768px）

* 卡片间距：16px

* 卡片比例：16:9 封面

**移动端：**

* 列数：2列

* 卡片间距：12px

**卡片内容：**

* 封面图（必选）

* 标题（最多2行，超出省略）

* UP主

* 时长

* 播放量（可选）

### 4.3 列表模式规格

**桌面端：**

* 行高：100px

* 左侧1/3：封面图

* 右侧2/3：标题、UP主、时长、日期、简介

**移动端：**

* 紧凑型布局

* 封面图 + 标题 + 时长

### 4.4 筛选组件

**触发方式：** 下拉菜单 / 侧边栏筛选面板

**筛选条件：**

| 维度 | 选项                             |
| -- | ------------------------------ |
| 时长 | 全部、0-5分钟、5-10分钟、10-30分钟、30分钟以上 |
| 时间 | 全部、最近一周、最近一月、最近一年              |
| 排序 | 最新发布、最早发布、最多播放、最少播放            |

**视觉反馈：**

* 已选筛选条件高亮显示

* 筛选结果数量实时更新

* 无结果时显示空状态引导

## 六、技术实现建议

### 6.1 技术选型

| 功能   | 推荐方案                         |
| ---- | ---------------------------- |
| 视图切换 | React State + CSS 过渡动画       |
| 网格布局 | CSS Grid 或 react-masonry-css |
| 无限滚动 | react-intersection-observer  |
| 筛选状态 | URL参数（支持分享）+ 本地状态            |

### 6.2 性能考虑

* 网格模式使用虚拟列表（窗口化）处理大量数据

* 图片使用懒加载

* 视图切换不重新请求数据

* 筛选结果缓存

### 6.3 兼容性

* 支持 Chrome、Firefox、Safari、Edge 最新版本

* 移动端响应式适配

* 保留无JavaScript降级方案（基础HTML展示）

## 七、验收标准

### 7.1 功能验收

| 验收项  | 通过标准                |
| ---- | ------------------- |
| 视图切换 | 三种模式可自由切换，动画流畅      |
| 网格模式 | 桌面端3-4列，移动端2列，响应式正常 |
| 列表模式 | 信息展示完整，布局紧凑         |
| 筛选功能 | 筛选条件生效，结果正确         |
| 排序功能 | 排序正确反映在视频顺序上        |

| 指标    | 目标值           |
| ----- | ------------- |
| 首次加载  | < 2s（视视频数量而定） |
| 视图切换  | < 300ms       |

### 7.3 用户体验验收

* 新用户能快速理解三种模式的差异

* 移动端触控友好

## 八、风险与约束

### 8.1 技术风险

* 大量视频（>100）时的性能表现需测试

* 不同屏幕尺寸的适配复杂度

### 8.2 时间约束

* MVP版本：视图切换 + 基础筛选

* V2版本：高级筛选 + 无限滚动 + 视频预览

### 8.3 后续迭代方向

* 视频预览（悬停播放GIF/短视频）

* 批量操作（批量收藏、批量删除）

* 数据分析（用户浏览习惯）

***

## 九、总结

本次优化的核心目标是**提升用户在大量视频场景下的浏览效率**，通过引入视图模式切换机制，让用户可以根据自己的浏览目的（回忆/发现/查找）选择最合适的展示方式。

**MVP范围：**

1. 视图模式切换（时光轴/网格/列表）
2. 基础筛选（时长/时间/排序）
3. 用户偏好记忆

**预期收益：**

* 用户浏览效率提升 50%+

* 单页信息密度提升 2-3倍

* 用户满意度提升

## TDD实施阶段

### 第一阶段：补充缺失的单元测试（红-绿-重构）

#### 1.1 HeroSection 组件测试

**用户旅程**: 作为用户，我希望在Hero区域看到与主题匹配的内容，以便了解当前模式

**测试用例**:

* 血怒模式渲染测试（标题、副标题、进度条）

* 混躺模式渲染测试（标题、副标题、进度条）

* 主题差异化内容测试

* CTA按钮渲染测试

* 头像和装饰元素测试

#### 1.2 CanteenHall 组件测试

**用户旅程**: 作为用户，我希望能搜索和筛选视频，以便快速找到想看的内容

**测试用例**:

* 组件渲染测试

* 搜索功能测试（按标题、标签搜索）

* 分类筛选测试

* 搜索结果计数显示测试

* 空状态显示测试

* 视频卡片渲染测试（缩略图、标题、标签、时长）

* 视频点击回调测试

* 清除搜索测试

#### 1.3 CVoiceArchive 组件测试

**用户旅程**: 作为用户，我希望浏览不同主题的语录，以便感受C皇的魅力

**测试用例**:

* 组件渲染测试

* 血怒/混躺主题语录显示测试

* 分类标签测试

* 语音播放按钮测试

* 特色语录显示测试

#### 1.4 VideoModal 组件测试

**用户旅程**: 作为用户，我希望点击视频后看到详细信息，以便决定是否观看

**测试用例**:

* 模态框渲染测试

* 视频信息显示测试（标题、描述、标签）

* 关闭按钮测试

* B站链接测试

* 主题样式测试

#### 1.5 YuxiaocPage 集成测试

**用户旅程**: 作为用户，我希望页面整体功能正常，主题切换流畅

**测试用例**:

* 页面渲染测试

* 主题切换功能测试

* 视频弹窗流程测试

* 子组件集成测试

## 执行命令

```bash
# 运行所有测试
npm test

# 运行单元测试
npm run test:unit

# 运行覆盖率报告
npm run test:coverage

# 运行E2E测试
npm run test:e2e

# 开发模式（监听）
npm run test:watch
```
